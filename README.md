# 食物分享爱 🍜❤️

> 情侣专属点餐微信小程序——一方点单，另一方做饭，记录每一次爱的味道。

[![微信小程序](https://img.shields.io/badge/微信-小程序-brightgreen)](https://mp.weixin.qq.com/)
[![云开发](https://img.shields.io/badge/云开发-已启用-blue)](https://cloud.weixin.qq.com/)
[![License](https://img.shields.io/badge/license-MIT-orange)](LICENSE)

## ✨ 功能特色

### 核心功能
- 🔗 **情侣绑定**：通过识别码快速绑定伙伴账号
- 📋 **菜单管理**：自定义菜品分类和菜品信息，支持图片/Emoji
- 🛒 **智能点单**：一方点单，另一方接单制作，智能分配掌勺人
- ✅ **订单管理**：简化状态流程（正在烹饪→已完成），用餐评价
- 📅 **餐饮日记**：日历视图记录每次用餐，周/月视图切换
- 🎮 **互动游戏**：决定今天谁掌勺的小游戏
- 🍽️ **今日推荐**：智能推荐荤素搭配的每日菜单（荤菜+素菜+汤品/甜点）

### 特色亮点
- 💡 **角色权限**：点单者评价，做饭者完成，分工明确
- 🎯 **智能推荐**：基于菜品分类的营养均衡推荐算法
- 📥 **快速导入**：一键导入示例菜单，快速上手
- 🎨 **精美设计**：粉橙渐变色调，圆角卡片，温馨浪漫
- 🚀 **性能优化**：数据缓存、页面预加载、即时反馈
- ⚙️ **个性设置**：支持修改头像、昵称，意见反馈

## 💾 云数据库设计

| 集合名 | 核心字段 | 说明 |
|--------|---------|------|
| `users` | openid, nickname, avatar, code, partnerId, isTestAccount | 用户信息，识别码用于绑定 |
| `categories` | name, icon, sort | 菜品分类 |
| `dishes` | name, desc, image, emoji, category, categoryId, sort | 菜品信息（支持图片和Emoji，category: meat/vegetable/soup/dessert） |
| `orders` | dishes[], note, status, orderedBy, creatorName, creatorAvatar, partnerId, createdAt, doneAt | 订单记录（status: cooking/done） |
| `reviews` | orderId, rating, comment, createdAt | 订单评价 |
| `feedbacks` | content, contact, userOpenid, userName, userAvatar, createdAt, status | 用户反馈 |

### 权限设置建议
- `users`：所有用户可读，仅创建者可写
- `categories`、`dishes`：所有用户可读写（情侣共享）
- `orders`、`reviews`：所有用户可读写

## 🚀 快速开始

### 1. 克隆项目
```bash
git clone https://github.com/yourusername/food-share-love.git
cd food-share-love
```

### 2. 配置云开发
1. 在[微信公众平台](https://mp.weixin.qq.com/)注册小程序
2. 开通云开发，创建云环境
3. 修改 `app.js` 中的云环境 ID：
```javascript
wx.cloud.init({
  env: 'your-env-id', // 替换为你的云环境 ID
  traceUser: true,
})
```

### 3. 上传云函数
1. 打开微信开发者工具
2. 右键 `cloudfunctions/login` → 上传并部署
3. 右键 `cloudfunctions/bindPartner` → 上传并部署
4. 右键 `cloudfunctions/deleteDish` → 上传并部署

### 4. 配置数据库
在云开发控制台创建以下集合：
- `users`
- `categories`
- `dishes`
- `orders`
- `reviews`

### 5. 运行项目
1. 在微信开发者工具中打开项目
2. 填写 AppID
3. 点击"编译"运行

## 🎯 核心功能实现

### 情侣绑定机制
- 每个用户注册时生成唯一的6位识别码
- 使用 `crypto.randomInt()` 生成密码学安全的随机码
- 通过云函数 `bindPartner` 实现双向绑定

### 今日推荐算法
- 从数据库查询所有菜品，按类别分组（荤菜/素菜/汤品/甜点）
- 智能推荐组合：1道荤菜 + 1道素菜 + 1道汤品/甜点
- 每次进入首页随机生成新推荐，营养均衡
- 支持数据库为空时的fallback机制

### 智能掌勺人分配
- 查询今天的订单记录
- 如果今天已有订单：显示"今天由 [对方名称] 掌勺"
- 如果今天还没订单：显示"今天由谁掌勺？待定"
- 下单后自动分配掌勺人（下单人的伙伴）

### 订单状态简化
- 简化为2个状态：`cooking`（正在烹饪）、`done`（已完成）
- 下单后立即进入"正在烹饪"状态
- 双方确认后进入"已完成"状态
- 移除了"等待接单"状态，简化流程

### 订单权限控制
- 点单者：只能评价，不能完成订单
- 做饭者：只能完成订单，不能评价
- 通过 `order.orderedBy` 判断角色

### 数据同步
- 使用 `_.or()` 查询实现订单双向可见
- 日记页面缓存订单日期数据（1分钟）
- 性能优化：页面预加载、震动反馈
