# 食分爱 — 完整部署操作指南

> 本文档从零开始，逐步说明如何在微信开发者工具中完成云开发配置、数据库初始化、云函数部署，以及如何验证小程序完整功能。

---

## 前置准备

- 已安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)（推荐最新稳定版）
- 已拥有微信小程序账号（AppID：见 `project.config.json`）
- 该小程序账号已开通**微信云开发**功能

---

## 第一步：导入项目

1. 打开微信开发者工具
2. 点击左上角「**+**」→「导入项目」
3. 选择项目目录：`food-share-love/`（即本仓库根目录）
4. AppID 填入 `project.config.json` 中的 `appid` 值
5. 点击「确定」，项目加载完成

---

## 第二步：创建云开发环境

> 如果你已有云开发环境并已在 `app.js` 中填写了环境 ID，可跳过此步。

1. 在开发者工具顶部工具栏点击「**云开发**」按钮（或 `Ctrl+Shift+C`）
2. 首次进入会提示创建环境，点击「**新建环境**」
3. 填写环境名称（如 `food-share-love-prod`），选择「**按量计费**」（有免费额度）
4. 等待环境初始化完成（约1-2分钟）
5. 进入环境后，复制右上角的**环境 ID**（格式类似 `food-share-xxx-xxxx`）

**更新 app.js 中的环境 ID：**

打开 `app.js` 第9行，将占位符替换为你的实际环境 ID：

```js
// 修改前
env: 'prod-env-id',

// 修改后（示例）
env: 'food-share-xxx-12345678',
```

---

## 第三步：创建云数据库集合

在云开发控制台左侧点击「**数据库**」，依次创建以下5个集合（点击「**+集合**」按钮）：

| 集合名 | 权限设置 | 说明 |
|--------|---------|------|
| `users` | 仅创建者可读写 | 用户信息 |
| `categories` | 所有用户可读，仅创建者可写 | 菜品分类 |
| `dishes` | 所有用户可读，仅创建者可写 | 菜品数据 |
| `orders` | 仅创建者可读写 | 订单数据 |
| `reviews` | 仅创建者可读写 | 订单评价 |

**设置权限方法：**

1. 点击集合名 → 右上角「**权限设置**」
2. 选择对应权限模板，点击「确认」

> **注意**：`categories` 和 `dishes` 建议设为「**所有用户可读，仅创建者可写**」，这样双方都能看到对方添加的菜品。

---

## 第四步：部署云函数

项目包含两个云函数，需要分别上传部署。

### 4.1 部署 login 云函数

1. 在开发者工具左侧文件树，展开 `cloudfunctions/`
2. 右键点击 `login` 文件夹
3. 选择「**上传并部署：云端安装依赖（不上传 node_modules）**」
4. 等待底部提示「上传成功」

### 4.2 部署 bindPartner 云函数

1. 同上，右键点击 `bindPartner` 文件夹
2. 选择「**上传并部署：云端安装依赖**」
3. 等待上传成功

### 4.3 验证云函数部署状态

1. 打开云开发控制台 → 左侧「**云函数**」
2. 确认 `login` 和 `bindPartner` 均显示为绿色「**已部署**」状态

---

## 第五步：配置云存储权限

云存储用于保存用户头像和菜品图片。

1. 云开发控制台 → 左侧「**存储**」
2. 点击右上角「**权限设置**」
3. 选择「**所有用户可读，仅创建者可写**」
4. 点击「确认」

---

## 第六步：添加静态资源

项目需要以下本地图片资源（tabBar 图标和占位图）。

### 6.1 创建目录结构

```
assets/
├── icons/
│   ├── home.png            (24×24 px，灰色)
│   ├── home-active.png     (24×24 px，#FF6B6B 主色)
│   ├── menu.png
│   ├── menu-active.png
│   ├── diary.png
│   ├── diary-active.png
│   ├── order.png
│   ├── order-active.png
│   ├── profile.png
│   └── profile-active.png
├── default-avatar.png      (圆形头像占位，建议 200×200 px)
└── default-dish.png        (菜品占位图，建议 400×300 px)
```

**图标建议：**
- 可从 [iconfont.cn](https://www.iconfont.cn) 搜索并下载 PNG 格式图标
- 尺寸推荐：**81×81 px**（微信 tabBar 推荐尺寸）
- 灰色版 `#999999`，激活版 `#FF6B6B`

> **临时方案**：如果暂时没有图标，可在 `app.json` 的 tabBar 配置中，先将 `iconPath` 和 `selectedIconPath` 字段留空或指向同一张占位图，不影响功能调试。

## 第七步：初始化菜品数据（可选）

首次使用时，菜单页面会是空的。可以通过两种方式添加数据：

### 方式 A：从小程序内手动添加

1. 运行小程序 → 进入「菜单」页面
2. 点击右上角「管理」进入编辑模式
3. 点击「+ 新增分类」，填写分类名（如：家常菜、汤类、主食）
4. 点击「+ 新增菜品」，填写菜品名称、简介、上传图片

### 方式 B：云数据库直接导入

在云开发控制台「数据库」页面，点击 `categories` 集合 → 「**添加记录**」：

```json
{
  "name": "家常菜",
  "sort": 1
}
```

点击 `dishes` 集合 → 「**添加记录**」：

```json
{
  "name": "红烧肉",
  "desc": "入口即化，肥而不腻",
  "image": "",
  "categoryId": "<上方创建的分类_id>",
  "sort": 1
}
```

## 第八步：功能联调测试

### 8.1 两个账号互相绑定

> 需要使用两部手机（或开发者工具 + 真机预览）分别登录。

**账号 A（点单方）：**
1. 进入「我的」页面
2. 记下「识别码」（6位大写字母+数字，如 `AB3C9D`）

**账号 B（做饭方）：**
1. 进入「我的」页面
2. 在「绑定另一半」输入框填入账号 A 的识别码
3. 点击「绑定」，提示「绑定成功」
4. 账号 A 刷新个人页，双方均可看到对方头像和昵称

### 8.2 验证完整下单流程

1. **账号 A**：进入「菜单」页面
2. 选择菜品，点击「+」增加数量
3. 点击右下角购物车按钮，进入购物车页
4. 填写备注（可选），点击「确认下单」
5. 看到下单成功页面，显示订单摘要

6. **账号 B**：切换到「订单」页面
7. 应看到账号 A 下单的订单（状态：待处理）
8. 点击「✓ 完成」，确认做好了
9. 订单状态变为「已完成」

### 8.3 验证日记页

1. 下单后，切换到「日记」页面
2. 今日日期上方应显示橙色小圆点
3. 点击今天，下方显示当日订单详情

### 8.4 验证洗碗游戏

1. 进入首页，点击「决定洗碗」
2. **骰子模式**：点击己方骰子摇动 → 再点击对方骰子摇动 → 点「看结果」
3. **剪刀石头布**：选择出拳 → 点「帮TA选」 → 查看结果

### 8.5 验证菜品管理

1. 菜单页点击「管理」进入编辑模式
2. 点击「+ 新增菜品」跳转编辑页
3. 点击图片区域上传图片（会上传至云存储）
4. 填写名称、简介、选择分类
5. 点击「添加菜品」，返回菜单页刷新确认

## 常见问题

**Q：云函数调用报错「cloud.callFunction:fail`**

A：检查 `app.js` 中的环境 ID 是否已替换，且云函数是否已成功部署（云开发控制台确认「已部署」状态）。

---

**Q：tabBar 图标显示不出来 / 路径报错**

A：确认 `assets/icons/` 目录下已放置对应 PNG 文件，且文件名与 `app.json` 中配置的路径完全一致（包括大小写）。

---

**Q：两账号绑定后，对方信息不显示**

A：首页和菜单页的伙伴信息在 `onShow` 生命周期加载。重新切换 tab 或下拉刷新即可。

---

**Q：图片上传失败**

A：检查云存储权限是否已设置为「所有用户可读，仅创建者可写」；确认手机有相册 / 相机授权。

---

**Q：数据库查询报错「permission denied`**

A：检查对应集合的权限配置。`orders` 和 `users` 应为「仅创建者可读写」；`categories` 和 `dishes` 建议「所有用户可读，仅创建者可写」。

---

## 数据库索引建议（性能优化）

在云开发控制台为以下字段创建索引：

| 集合 | 字段 | 类型 |
|------|------|------|
| `users` | `code` | 单字段，唯一 |
| `users` | `openid` | 单字段，唯一 |
| `dishes` | `categoryId` | 单字段 |
| `orders` | `createdAt` | 单字段，降序 |

**创建方法**：云开发控制台 → 数据库 → 选中集合 → 「**索引管理**」→「添加索引」

## 发布上线

1. 开发者工具顶部 → 「**上传**」代码（填写版本号和备注）
2. 登录 [微信公众平台](https://mp.weixin.qq.com) → 「版本管理」→「提交审核」
3. 审核通过后点击「发布」

> 上线前确认：所有云函数已部署、数据库权限已配置、tabBar 图标资源齐全。
