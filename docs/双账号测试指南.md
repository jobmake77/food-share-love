# 食物分享爱 - 双账号测试指南

## 📱 测试准备

### 设备要求
- **方案1（推荐）**：两台手机，分别登录不同的微信账号
- **方案2**：一台手机 + 一台电脑（使用微信开发者工具）
- **方案3**：两台电脑，都使用微信开发者工具

### 测试环境
- 微信开发者工具版本：最新稳定版
- 小程序基础库：2.2.3 或以上
- 云开发环境：已配置并启用

---

## 🚀 测试流程

### 第一步：初始化两个账号

#### 账号 A（用户1）
1. 打开小程序
2. 首次进入会自动登录并生成识别码
3. 进入"我的"页面
4. 记录识别码（例如：`ABC123`）
5. 设置昵称为"用户A"
6. 上传头像（可选）

#### 账号 B（用户2）
1. 在另一台设备上打开小程序
2. 首次进入会自动登录并生成识别码
3. 进入"我的"页面
4. 记录识别码（例如：`XYZ789`）
5. 设置昵称为"用户B"
6. 上传头像（可选）

---

### 第二步：绑定伙伴关系

#### 在账号 A 上操作
1. 进入"我的"页面
2. 找到"绑定另一半"区域
3. 输入账号 B 的识别码：`XYZ789`
4. 点击"绑定"按钮
5. ✅ 预期结果：显示"绑定成功！"
6. ✅ 预期结果：页面显示"用户B"的头像和昵称

#### 在账号 B 上验证
1. 返回首页或刷新"我的"页面
2. ✅ 预期结果：自动显示"用户A"为伙伴
3. ✅ 预期结果：无需再次输入识别码

## 🧪 功能测试清单

### 1. 菜单管理测试

#### 测试场景 1.1：账号 A 创建菜单
**操作步骤：**
1. 账号 A 进入"菜单"页面
2. 点击右上角"编辑"按钮
3. 点击"新增分类"，创建分类"川菜"
4. 点击"+"添加菜品
   - 菜品名称：宫保鸡丁
   - 描述：麻辣鲜香
   - 分类：川菜
5. 保存菜品

**预期结果：**
- ✅ 账号 A 能看到新创建的菜品
- ✅ 账号 B 也能看到这个菜品（刷新菜单页面）
- ✅ 两个账号看到的菜单是同步的

#### 测试场景 1.2：账号 B 编辑菜品
**操作步骤：**
1. 账号 B 进入"菜单"页面
2. 点击"编辑"按钮
3. 点击"宫保鸡丁"进入编辑
4. 修改描述为"麻辣鲜香，超级好吃"
5. 保存

**预期结果：**
- ✅ 账号 B 能成功编辑
- ✅ 账号 A 刷新后能看到修改

#### 测试场景 1.3：导入示例菜单
**操作步骤：**
1. 任一账号进入"我的"页面
2. 点击"✨ 导入示例菜单"
3. 查看预览信息
4. 点击"一键导入"

**预期结果：**
- ✅ 导入成功，显示 5 个分类，18 道菜品
- ✅ 两个账号都能看到导入的菜单

## 🐛 常见问题排查

### 问题 1：看不到对方下的单
**排查步骤：**
1. 检查是否已经绑定伙伴
2. 检查云数据库权限设置
3. 检查订单查询逻辑（是否使用了 `_.or()` 查询）
4. 刷新页面重试

### 问题 2：绑定失败
**排查步骤：**
1. 检查识别码是否正确（6位大写字母+数字）
2. 检查网络连接
3. 检查云函数 `bindPartner` 是否部署
4. 查看控制台错误信息

### 问题 3：菜单不同步
**排查步骤：**
1. 检查云数据库权限（应该是"所有用户可读写"或"仅创建者可写"）
2. 刷新页面
3. 检查是否在同一个云环境

### 问题 4：删除菜品失败
**排查步骤：**
1. 检查云函数 `deleteDish` 是否部署
2. 检查是否是菜品的创建者
3. 查看控制台错误信息

## 🎯 测试建议

1. **按顺序测试**：先测试基础功能（登录、绑定），再测试核心功能（点单、完成）
2. **交叉测试**：A→B 和 B→A 都要测试
3. **记录问题**：发现问题立即记录，包括复现步骤
4. **清理数据**：测试完成后可以清理测试数据，或者使用"创建测试伙伴"功能
5. **真实场景**：模拟真实使用场景，比如早中晚不同时间点单
