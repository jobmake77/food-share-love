# 发布体验版测试指南

## 📦 发布体验版

### 1. 上传代码
1. 打开微信开发者工具
2. 点击右上角"上传"按钮
3. 填写版本号（例如：`1.0.0`）
4. 填写项目备注（例如：`双账号测试版本`）
5. 点击"上传"

### 2. 设置体验版
1. 登录[微信公众平台](https://mp.weixin.qq.com/)
2. 进入"开发管理" → "开发版本"
3. 找到刚上传的版本
4. 点击"选为体验版"

### 3. 添加体验成员
1. 在"成员管理" → "体验成员"中
2. 添加你的两个微信号（扫码或输入微信号）
3. 体验成员上限：40人

### 4. 获取体验版二维码
1. 在"开发版本"页面
2. 点击"体验版"旁边的二维码图标
3. 保存二维码图片

### 5. 开始测试
1. 用第一个微信扫码进入小程序
2. 用第二个微信扫码进入小程序
3. 现在可以用两个真实账号测试了！

---

## 🔧 快速测试方案（单账号）

如果暂时只有一个账号，可以使用"创建测试伙伴"功能：

1. 进入"我的"页面
2. 在"绑定另一半"区域
3. 点击"创建测试伙伴账号"按钮
4. 系统会自动创建虚拟伙伴并完成绑定

**注意：** 上线前记得把 `pages/profile/profile.js` 中的 `showDevMode` 改为 `false`

---

## ⚠️ 不建议直接发正式版的原因

1. **无法回滚**：正式版发布后，所有用户都会更新
2. **影响用户**：如果有 bug，会影响真实用户体验
3. **审核时间**：正式版需要微信审核（1-7天）
4. **测试不充分**：没有经过完整测试就上线风险大

---

## 📋 体验版 vs 正式版对比

| 特性 | 体验版 | 正式版 |
|------|--------|--------|
| 需要审核 | ❌ 不需要 | ✅ 需要（1-7天） |
| 谁能访问 | 仅体验成员 | 所有用户 |
| 更新速度 | 立即生效 | 审核通过后 |
| 测试功能 | ✅ 完整功能 | ✅ 完整功能 |
| 云开发 | ✅ 可用 | ✅ 可用 |
| 回滚 | ✅ 随时更新 | ❌ 需重新审核 |

---

## 🎯 推荐测试流程

1. **本地开发** → 真机调试（单账号快速验证）
2. **体验版** → 双账号完整测试
3. **正式版** → 测试通过后发布

---

## 💡 体验版测试技巧

### 快速更新体验版
每次修改代码后：
1. 点击"上传"
2. 选为体验版
3. 体验成员刷新小程序即可看到更新

### 查看体验版日志
1. 微信公众平台 → "开发管理" → "运维中心"
2. 可以看到错误日志和性能数据

### 体验版限制
- 最多 40 个体验成员
- 功能与正式版完全一致
- 可以访问云开发环境

---

## 🚀 发布正式版（测试通过后）

1. 在"开发版本"中点击"提交审核"
2. 填写审核信息
3. 等待审核（通常 1-3 天）
4. 审核通过后点击"发布"

---

**建议：先用体验版充分测试，确认没问题后再发布正式版！**
