# 意见反馈功能部署说明

## 功能说明

用户可以在个人页点击"意见反馈"进入反馈页面，输入反馈内容和联系方式（选填），提交后保存到云数据库 `feedbacks` 集合。开发者可以在云开发控制台查看所有用户反馈。

## 部署步骤

### 步骤 1：创建数据库集合

在云开发控制台创建 `feedbacks` 集合：

**集合结构**：
```json
{
  "_id": "自动生成",
  "content": "反馈内容",
  "contact": "联系方式（选填）",
  "userOpenid": "用户openid",
  "userName": "用户昵称",
  "userAvatar": "用户头像",
  "createdAt": "创建时间",
  "status": "pending/replied/closed"
}
```

**权限设置**：
- 读取：`doc._openid == auth.openid`（用户只能看自己的反馈）
- 写入：`false`（通过云函数写入）
- 创建：`false`（通过云函数创建）

### 步骤 2：上传云函数

在微信开发者工具中：

1. **上传 sendFeedback 云函数**：
   - 右键 `cloudfunctions/sendFeedback` 文件夹
   - 选择"上传并部署：云端安装依赖"

### 步骤 3：查看反馈

在云开发控制台查看用户反馈：

1. 进入云开发控制台
2. 点击"数据库"
3. 选择 `feedbacks` 集合
4. 查看用户提交的反馈

每条反馈包含：
- 用户昵称
- 反馈内容
- 联系方式（如果用户填写）
- 提交时间
- 状态（pending/replied/closed）

### 步骤 4：测试功能

1. **测试意见反馈**：
   - 进入个人页
   - 点击"意见反馈"
   - 输入反馈内容
   - 点击提交
   - 在云开发控制台查看是否保存成功

## 分享功能说明

"分享小程序"功能使用微信原生分享：
- 点击后会弹出微信分享面板
- 可以分享给微信好友或群聊
- 支持分享到朋友圈（需在 page.json 中配置）

分享内容可以在 `pages/profile/profile.js` 的 `onShareAppMessage` 方法中自定义。

## 文件清单

### 新增页面
- `subpages/feedback/` - 意见反馈页面

### 新增云函数
- `cloudfunctions/sendFeedback/` - 发送反馈

### 修改文件
- `pages/profile/profile.wxml` - 添加反馈入口和分享按钮
- `pages/profile/profile.js` - 添加分享配置
- `pages/profile/profile.wxss` - 添加按钮样式
- `app.json` - 注册反馈页面

## 注意事项

1. **反馈查看**：
   - 当前版本只保存到数据库
   - 需要手动在云开发控制台查看
   - 如需邮件通知，需要额外配置第三方邮件服务

2. **分享功能**：
   - 使用微信原生分享，无需额外开发
   - 可以自定义分享标题和封面图
   - 分享封面图需要准备图片资源

3. **数据管理**：
   - 建议定期查看和处理用户反馈
   - 可以修改反馈的 status 字段标记处理状态
   - pending: 待处理
   - replied: 已回复
   - closed: 已关闭
