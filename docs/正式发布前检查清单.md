# 正式发布前检查清单

> 适用于「食物分享爱」微信小程序上线前最终确认。  
> 建议在提交审核前 24 小时内完整打勾一次。

## A. 版本与代码

- [ ] `git status` 干净（无误提交/漏提交）
- [ ] 本次发布版本号、更新说明已确认
- [ ] 高风险调试代码已清理（敏感日志、临时代码、占位配置）

## B. 小程序配置

- [ ] `project.config.json` 的 `appid` 为正式小程序 AppID
- [ ] `app.js` 的 `wx.cloud.init({ env })` 为正式云环境
- [ ] `app.json` 页面路径、tabBar 图标路径均可访问

## C. 云开发资源

- [ ] 数据库集合已创建：`users`、`categories`、`dishes`、`orders`、`bind_requests`、`feedbacks`
- [ ] 数据库权限已复核（至少不暴露敏感用户数据）
- [ ] 云函数已部署并为最新代码：`login`、`bindPartner`、`deleteDish`、`sendFeedback`
- [ ] `sendFeedback` 中订阅消息模板 ID 已按生产配置完成（若使用）
- [ ] 云存储权限与套餐状态已确认（头像/菜品图上传可用）

## D. 合规与审核材料

- [ ] 微信公众平台「隐私保护指引」已按真实能力填写并发布
- [ ] 用户信息与用途说明（昵称头像、图片上传等）与产品实际一致
- [ ] 审核备注已准备：核心功能入口、测试路径、注意事项
- [ ] 小程序类目、服务内容与实际功能一致

## E. 发布前回归（双账号）

- [ ] 账号 A/B 绑定与解绑流程正常
- [ ] 菜单与分类：新增、编辑、删除、导入正常
- [ ] 点单链路：下单、接单完成、评价完整可走通
- [ ] 日记页可正确看到历史订单
- [ ] 反馈提交通路可用（提交成功且后台可见）
- [ ] 分享、设置、图片上传等周边功能正常

## F. 发布动作

- [ ] 微信开发者工具上传代码（版本号 + 备注）
- [ ] 公众平台提交审核，审核信息填写完整
- [ ] 审核通过后执行正式发布
- [ ] 发布后 30 分钟内进行线上冒烟检查（登录、点单、订单、反馈）

## G. 回滚预案（建议）

- [ ] 保留上一个稳定版本记录（版本号 + 提交号）
- [ ] 出现 P0/P1 问题时，明确应急流程：下线入口文案、紧急修复、重新提审
