# 测试模式使用指南

## 🎯 功能说明

测试模式允许你在单个账号下模拟双方操作，方便测试情侣互动功能。

## 📝 使用步骤

### 1. 创建测试伙伴

1. 进入**个人页**（Profile）
2. 找到"绑定另一半"区域
3. 点击**"创建测试伙伴账号"**按钮
4. 确认创建

系统会自动：
- 创建一个虚拟伙伴账号
- 完成双向绑定
- 开启测试模式

### 2. 切换身份

创建测试伙伴后，在个人页右上角会出现**"🔄 切换身份测试"**按钮。

点击按钮可以在两个身份之间切换：
- **你**：原始账号
- **测试伙伴**：虚拟账号

### 3. 测试场景

#### 场景1：点单流程测试

1. **以"你"的身份**：
   - 进入菜单页
   - 选择菜品加入购物车
   - 下单给"测试伙伴"

2. **切换到"测试伙伴"身份**：
   - 进入订单页
   - 查看收到的订单
   - 标记订单为"已完成"

3. **切换回"你"的身份**：
   - 进入订单页
   - 查看订单状态
   - 进行评价

#### 场景2：游戏功能测试

1. **掷骰子**：
   - 单人模式，无需切换身份
   - 点击按钮自动掷两次

2. **石头剪刀布**：
   - 单人模式，系统自动对战
   - 无需切换身份

3. **疯狂点点点**：
   - 双人模式，需要两个手机
   - 测试模式下可以一个人同时点两边

## ⚠️ 注意事项

1. **测试模式标记**：
   - 创建测试伙伴后，`app.globalData.isTestMode` 会设置为 `true`
   - 测试账号在数据库中有 `isTestAccount: true` 标记

2. **数据隔离**：
   - 测试伙伴的数据存储在真实数据库中
   - 可以在数据库中手动删除测试账号

3. **身份切换**：
   - 切换身份后会重新加载首页
   - 所有页面都会使用当前身份的数据

4. **退出测试模式**：
   - 解绑伙伴即可退出测试模式
   - 或者删除测试账号数据

## 🔧 开发者选项

在 `pages/profile/profile.js` 中：

```javascript
showDevMode: true  // 设置为 false 可以隐藏测试功能
```

上线前建议设置为 `false`，避免用户误操作。

## 💡 真实双账号测试

如果需要测试真实的双账号场景（如实时通知、消息推送等），建议：

1. **使用两个手机**：
   - 手机1：用你的微信登录
   - 手机2：用朋友的微信登录
   - 互相输入识别码绑定

2. **使用开发者工具 + 手机**：
   - 开发者工具：切换测试号
   - 手机：真实微信登录
   - 互相绑定测试

## 📊 测试清单

- [ ] 点单流程：下单 → 接单 → 完成 → 评价
- [ ] 订单可见性：双方都能看到订单
- [ ] 操作权限：下单者只能评价，厨师能标记完成
- [ ] 游戏功能：掷骰子、石头剪刀布、疯狂点点点
- [ ] 日记功能：查看历史订单和菜品
- [ ] 菜单管理：添加、编辑、删除菜品
- [ ] 绑定/解绑：识别码绑定、解绑功能
