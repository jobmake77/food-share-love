# 绑定确认机制更新说明

## 更新内容

### 1. 绑定流程改进
**之前**：A 输入 B 的识别码 → 直接绑定成功
**现在**：A 输入 B 的识别码 → 发送绑定请求 → B 确认 → 完成双向绑定

### 2. 新增功能
- ✅ 绑定请求确认机制
- ✅ 待确认请求列表显示
- ✅ 接受/拒绝绑定请求
- ✅ 修复测试伙伴双向绑定问题

### 3. 数据库变更
需要在云开发控制台创建新集合：`bind_requests`

**集合结构**：
```json
{
  "_id": "自动生成",
  "fromId": "发起者用户ID",
  "fromName": "发起者昵称",
  "fromAvatar": "发起者头像",
  "targetId": "目标用户ID",
  "status": "pending",
  "createdAt": "创建时间"
}
```

**权限设置**（在云开发控制台设置）：
- 读取：`doc.targetId == auth.openid || doc.fromId == auth.openid`
- 写入：`doc.fromId == auth.openid`
- 创建：`true`
- 更新：`false`
- 删除：`doc.targetId == auth.openid || doc.fromId == auth.openid`

## 部署步骤

### 1. 上传云函数
在微信开发者工具中：
1. 右键点击 `cloudfunctions/bindPartner` 文件夹
2. 选择"上传并部署：云端安装依赖"

### 2. 创建数据库集合
在云开发控制台：
1. 进入"数据库"
2. 点击"添加集合"
3. 集合名称：`bind_requests`
4. 点击"权限设置"，按照上面的权限规则配置

### 3. 测试流程
1. 用户 A 在个人页输入用户 B 的识别码，点击"发送请求"
2. 用户 B 在个人页看到待确认的绑定请求
3. 用户 B 点击"接受"或"拒绝"
4. 接受后，双方完成绑定

## 测试模式说明
- 创建测试伙伴时，会自动完成双向绑定（跳过确认流程）
- 切换身份后，可以看到双方的绑定状态
- 测试伙伴账号会标记 `isTestAccount: true`

## 注意事项
1. 旧的绑定数据不受影响，已绑定的用户无需重新绑定
2. 新用户绑定时需要对方确认
3. 绑定请求会按时间倒序显示
4. 拒绝请求后，发起者可以重新发送
